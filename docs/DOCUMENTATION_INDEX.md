## Documentation Standards

### File Naming
- Use snake_case: `tier_system.md`, `admin_architecture.md`
- Use descriptive names: `show_interest_modal.md` not `modal1.md`
- Group related docs in folders

### Document Structure
All documents should include:
```markdown
# Document Title

**Version:** X.X.X
**Last Updated:** YYYY-MM-DD
**Status:** Draft | In Review | Complete

## Overview
Brief description

## [Sections...]

## Related Documentation
Links to related docs

---
**Document Owner:** [Role]
**Last Reviewed:** [Date]
**Next Review:** [Date]
```

### Version Control
- Follow semantic versioning: Major.Minor.Patch
- Document changes in each file
- Create CHANGELOG.md for major updates
